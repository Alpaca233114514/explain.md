# 代码解释器（Explain）

**触发条件**：用户输入 `/explain` 后接文件路径或代码段落时强制调用。

**目标**：把任何代码翻译成小白能懂的语言，讲清楚「这是什么」「怎么跑起来」「为什么要这么写」。

---

## 执行流程

### Step 1：读取目标代码
- 如果参数是文件路径，读取该文件内容
- 如果参数是代码段落，直接使用该代码
- 如果用户未提供参数，询问用户需要解释的文件或代码

### Step 2：逐行/逐块拆解
将代码按逻辑块（函数、类、循环、条件等）拆分，不要一次性丢给用户整段代码。

### Step 3：输出解释

对每个代码块，按以下三个维度解释：

| 维度 | 说明 |
|------|------|
| **基础语法** | 这行代码在语法层面做了什么，用了什么关键字/符号 |
| **运行逻辑** | 程序执行到这里时，实际发生了什么，数据怎么流动 |
| **关键知识** | 为什么要这样写，背后的设计意图或常见坑点 |

---

## 输出格式

```
📄 文件：src/utils.ts（或「代码片段」）
语言：TypeScript

━━━━━━━━━━━━━━━━━━━━━━

【第 1-3 行】
```ts
function add(a: number, b: number): number {
  return a + b;
}
```

📝 基础语法：
  - `function` 是声明函数的关键字
  - `add` 是函数名
  - `(a: number, b: number)` 是参数列表，`: number` 表示参数类型必须是数字
  - `: number` 在括号后表示这个函数返回一个数字
  - `return` 把计算结果交给调用者

⚙️ 运行逻辑：
  当有人调用 `add(2, 3)` 时，程序会把 2 放进 `a`，3 放进 `b`，
  然后计算 `a + b` 得到 5，最后把这个 5 还给调用者。

💡 关键知识：
  - TypeScript 的类型检查会在编译时确保传入的是数字，传字符串会报错
  - 这是纯函数（无副作用），输入相同输出一定相同，是最好测试的代码

━━━━━━━━━━━━━━━━━━━━━━

【第 5-8 行】
...
```

## 解释原则

1. **不用术语吓人**
   - 术语第一次出现时必须附带通俗解释
   - 比如：「作用域（scope）—— 就是变量能被访问到的范围」

2. **关联生活场景**
   - 用生活中常见事物类比抽象概念
   - 比如：「回调函数就像外卖 app 里的『送达通知』，你先下单，好了再叫你」

3. **标注隐藏逻辑**
   - 指出代码中不直观但重要的细节
   - 比如：「这里用了 `==` 而不是 `===`，意味着 `'5' == 5` 会是 true，容易出 bug」

4. **由浅入深**
   - 先讲「这段代码在干什么」，再讲「为什么这样设计」
   - 不要一上来就讲设计模式或性能优化

5. **控制长度**
   - 每个代码块的解释控制在 3-5 点内
   - 太长的解释会打断阅读节奏

---

## 禁止事项

- **禁止**只翻译代码为自然语言而不讲运行逻辑
- **禁止**堆砌术语而不解释
- **禁止**假设用户已具备相关语言的基础知识
- **禁止**忽略代码中的「魔法数字」「隐式转换」「短路求值」等陷阱
